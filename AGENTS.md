# Rules

## プロジェクト目的 (Purpose)
対戦中の League of Legends (LoL) の試合において、敵チームの中で最も育っている（最もキルを獲得しキャリーしている）警戒すべきチャンピオンをリアルタイムで特定し、ゲーム画面上に常に表示しておくことで、プレイヤーが警戒対象を瞬時に把握できるようにする。

## サービス構成・レイヤ (Architecture)
1. **データ取得レイヤ (`src/data_fetcher.py`)**:
   - ローカルで稼働する LoL クライアントの Live Client API (`https://127.0.0.1:2999/liveclientdata/allgamedata`) をポーリング。
   - 試合中の全プレイヤー情報から敵チームメンバーのみを抽出し、リアルタイムKDA（整数）を基に強さ評価スコアを算出して最強の敵を特定する。
2. **UI表示レイヤ (`src/overlay.py`)**:
   - Tkinter を用いた、枠なしで常に最前面表示（Topmost）されるスタイリッシュなダークモード調オーバーレイ。
   - 画面左上に配置され、左ドラッグによる任意の場所への移動や、右クリック / Escapeキーでの安全な終了に対応。
3. **メイン制御ループ (`src/main.py`)**:
   - データ取得とUI更新のライフサイクルを制御するエントリーポイント。

## 開発ルール (Project Rules)
- LoL の Live Client API を定期的にポーリングし、敵の中で最もKDAの高いチャンピオンをリアルタイムで特定する。
- 画面左上に Tkinter を使用した美しいダークモード調のドラッグ可能・枠なしオーバーレイウィンドウを表示し、その敵チャンピオンの名前とリアルタイムのKDA（整数表記、例: `5 / 1 / 2`）を常に最前面に表示する。
- ゲームが起動していない、あるいはロード中の場合は、待機状態 (`Waiting for game...`) として表示をリセットする。
- プロジェクトファイルはワークスペース内に配置する。
- **Gitコミットメッセージは必ず日本語で記述する。**
